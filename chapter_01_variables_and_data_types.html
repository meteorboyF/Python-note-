📦 1. Variables

A variable is a label that points to a value stored in memory. Python creates the variable the moment you assign a value — no explicit type declaration is needed (Python is dynamically typed).

variables_intro.py
message = "Hello world! This is Fardeen"
print(message)          # Hello world! This is Fardeen

# Reassign any time — Python adapts
message = "Hello again!"
print(message)          # Hello again!

# Multiple assignment on one line
x = y = z = 0
a, b, c = 1, 2, 3       # unpacking
⚠️
NameErrorMeans you either forgot to assign a value, or made a typo in the variable name. Python is case-sensitive: Message ≠ message.

📝 2. Variable Naming Rules & Conventions

  • Only letters, numbers, underscores — never start with a digit
  • Spaces are illegal — use underscore_case
  • Avoid Python keywords: for, if, class, print…
  • Be short but descriptive: student_name beats sn or full_student_name_string
  • Avoid confusing l (letter ell) and O (letter oh) — they look like 1 and 0
naming.py
# ✅ Valid
student_name = "Fardeen"
_internal = 42
item2 = "second"
MAX_SIZE = 100          # constants use ALL_CAPS (convention)

# ❌ Invalid
# 2items = "bad"        → SyntaxError
# my name = "bad"       → SyntaxError
# for = 10              → SyntaxError (keyword)
💡
PEP 8 ConventionUse snake_case for variables & functions, PascalCase for classes, ALL_CAPS for constants. This is the universal Pythonic style.

📜 4. Working with Strings

A string is an immutable sequence of characters. Use single ' or double " quotes — they are equivalent.

strings.py
"This is a string."
'This also works.'

# Use the other quote type to include quotes/apostrophes:
msg1 = "I told my friend, 'Python is great!'"
msg2 = "The language 'Python' is named after Monty Python."
msg3 = "One of Python's strengths is its diverse community."

# Triple quotes → multiline strings
bio = """My name is Fardeen.
I love Python programming.
It's amazing."""
🚨
SyntaxError — Mismatched Quotes'One of Python's strengths...' ← the apostrophe ends the string early. Fix: wrap in double quotes or escape: \'.
🔠 4.1 Case Transformation Methods
case_methods.py
name = "max verstappen"

print(name.title())      # Max Verstappen  (each word capitalised)
print(name.upper())      # MAX VERSTAPPEN
print(name.lower())      # max verstappen
print(name.capitalize()) # Max verstappen  (only first char)
print(name.swapcase())   # MAX VERSTAPPEN → lower and vice versa
💡
Why .lower() mattersWhen storing user input in a database, always .lower() it first — you can't trust what case the user typed. Display with whatever case makes sense.
🔗 4.2 String Concatenation & Repetition

✨ 4.3 F-Strings — Modern String Interpolation

Introduced in Python 3.6, f-strings are the preferred way to embed expressions in strings. Prefix with f and place variables/expressions in { }.

fstrings.py
first = "ada"
last  = "lovelace"
full  = f"{first} {last}"
print(f"Hello, {full.title()}!")     # Hello, Ada Lovelace!

# Any expression works inside { }
price, qty = 49.99, 3
print(f"Total: ${price * qty:.2f}")  # Total: $149.97

# Python 3.8+ self-documenting expressions
x = 42
print(f"{x = }")                     # x = 42

# Alignment / padding
print(f"{'Name':<12} {'Score':>6}")
print(f"{'Fardeen':<12} {98:>6}")
ℹ️
Older alternatives (Python < 3.6)"Hello, {}!".format(name) or "Hello, %s!" % name — both still valid but f-strings are faster and cleaner.

Python is strongly typed — it won't silently mix types. Use built-in functions to convert between them.

💡 8. Tips & Tricks

🔁 Swap Without Temp

Use tuple unpacking: a, b = b, a. No third variable needed — purely Pythonic.

🔍 Check Truthiness

Empty string "", 0, None, empty list [] are all falsy. Use if name: instead of if name != "":.

⛓️ Chain String Methods

email.strip().lower().replace(" ", "") — chain as many methods as you need in one line.

📐 Formatted Numbers

f"{1000000:,}" → 1,000,000. Use :,.2f for currency: f"${price:,.2f}"

🧮 Multiple Assignment

x = y = z = 0 assigns the same value to multiple variables at once.

🔤 String Check Shortcuts

"py" in "python" → True. Use in / not in for substring checks instead of .find().

📏 String Length Guard

Always validate user input length: if len(username) > 20: ... before saving to a DB.

🎯 Walrus Operator (3.8+)

if (n := len(name)) > 5: — assigns and checks in one expression. Great in loops.

tricks.py
# Swap
a, b = 1, 2
a, b = b, a   # a=2, b=1

# Truthiness check
username = input("Enter name: ").strip()
if not username:
    print("Name cannot be empty!")

# in operator for substring
if "gmail" in email.lower():
    print("Gmail user")

# Number formatting
salary = 95000
print(f"Salary: ${salary:,.2f}")   # Salary: $95,000.00

# Walrus operator
import random
nums = [random.randint(1,10) for _ in range(5)]
if (n := len(nums)) > 3:
    print(f"Long list: {n} items")

🎯 9. Interview Questions

Click each question to reveal the answer.

Q1: What is the difference between a variable and a constant in Python? ▼
Python has no built-in const keyword. Variables are mutable labels pointing to values. Constants are a convention — variables named in ALL_CAPS (e.g., MAX_SIZE = 100) signal to other developers "don't change this." Python won't prevent you from changing it; the convention is a contract between programmers.
Q2: What is the difference between is and ==? ▼
== checks value equality (do they hold the same value?). is checks identity (are they the exact same object in memory?). Example: a = [1,2]; b = [1,2] → a == b is True but a is b is False. Use is only for None comparisons: if x is None:.
Q3: Is Python dynamically or statically typed? What does that mean? ▼
Python is dynamically typed: types are determined at runtime, not compile time. You don't declare a variable's type. The same variable can hold different types at different times: x = 5 then x = "hello". Python is also strongly typed: it won't implicitly coerce types — "5" + 5 raises a TypeError.
Q4: What are truthy and falsy values in Python? ▼
Falsy values: False, 0, 0.0, "" (empty string), [], {}, (), None. Everything else is truthy. This lets you write clean guards: if username: instead of if username != "":.
Q5: What is the difference between str(), repr(), and format()? ▼
str(x) returns a human-readable string — what you'd show a user. repr(x) returns a developer-oriented string that could recreate the object (e.g., includes quotes for strings). format() is used for formatted output and is now largely replaced by f-strings.
Q6: Why does 0.1 + 0.2 ≠ 0.3 in Python? ▼
Floats use IEEE 754 binary floating-point representation, which cannot represent all decimal fractions exactly. 0.1 in binary is a repeating fraction, so small rounding errors accumulate. Fix: use round(0.1+0.2, 2) or the decimal module for exact arithmetic: from decimal import Decimal; Decimal("0.1") + Decimal("0.2").
Q7: What is the difference between int division in Python 2 vs Python 3? ▼
In Python 2, dividing two integers returns an integer with the remainder truncated: 3/2 → 1. In Python 3, division always returns a float: 3/2 → 1.5. To get floor division in Python 3, use //: 3//2 → 1.
Q8: Are strings mutable in Python? ▼
No. Strings are immutable — once created, their contents cannot be changed. Methods like .upper() or .strip() return a new string without modifying the original. To "modify" a string, you must reassign: s = s.upper().
Q9: What is a NameError vs a TypeError? ▼
NameError: Python can't find the variable — either it was never defined or there is a typo. TypeError: An operation is applied to the wrong type, e.g., trying to concatenate an int to a str without explicit conversion.
Q10: What is PEP 8 and why does it matter? ▼
PEP 8 is Python's official style guide, defining conventions for formatting code: indentation (4 spaces), max line length (79 chars), naming conventions (snake_case, PascalCase, ALL_CAPS), and more. Following PEP 8 makes code consistent, readable, and maintainable across teams.
